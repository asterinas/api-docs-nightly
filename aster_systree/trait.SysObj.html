fn name(&self) -> &SysStr

Returns the name of a node.

The name is guaranteed not to contain two special characters: '/' and '\0'.

The root node of a SysTree has an empty name. All other inodes must have an non-empty name.

Source

fn init_parent(&self, parent: Weak<dyn SysBranchNode>)

Initializes the parent in the SysTree.

An appropriate timing to call this method is when a SysTree node is added to another node as a child.

§Panics

This method should be called at most once; otherwise, it may trigger panicking.

fn path(&self) -> SysStr

Returns the path.

The path of a SysTree node is defined as below:

  • If a node is the root, then the path is /.
  • If a node is a non-root, we will need to see if it has a parent:
    • If the node has a parent, then the path is defined as the parent path and the node’s name joined with a / in between;
    • Otherwise, the node’s name is taken as its path.
