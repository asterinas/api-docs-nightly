The maximum bytes of all buffers in the I/O vector.

According to man pages, the kernel should fail with Errno::EINVAL if the number of bytes in the I/O vector exceeds this threshold. See https://man7.org/linux/man-pages/man2/writev.2.html.

However, the actual Linux behavior is to truncate the buffer and ignore the remaining buffer space. See https://elixir.bootlin.com/linux/v6.12.6/source/lib/iov_iter.c#L1463.

Typical 64-bit architectures do not have 64-bit virtual address space, and the value of MAX_IO_VECTOR_LENGTH is relatively small. Therefore, userspace may not be able to supply a valid I/O vector containing so many bytes. Nevertheless, we should still check against this to prevent overflows in the future, e.g., when the virtual address space becomes larger.
