An independent allocation domain within an ext2 filesystem.

An ext2 volume is partitioned into equally sized block groups so that related blocks and inodes can be allocated close together on disk, reducing seek distance and improving locality.

§Invariants

  • All group-local bitmap indices fit in u16 because the superblock enforces nr_blocks_per_group <= block_size * 8 (max 32768 for 4 KiB blocks), and nr_inodes_per_group is similarly bounded.
  • The group descriptor and both bitmaps are protected by a single RwMutex. Block and inode allocation/free update bitmaps and group counters in one critical section.
  • Metadata block pointers (block bitmap, inode bitmap, inode table) are validated at mount time and are guaranteed to lie within the group’s block range and be marked as allocated.

§Locking

BlockGroup uses two independent locks:

  • metadata — protects the group descriptor and both allocation bitmaps. Held briefly during alloc/free operations.
  • inode_cache — protects the per-group live inode map. Uses double-checked locking (read then promote to write on miss).
