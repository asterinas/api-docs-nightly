Expand description

Common operations shared by all namespace types.

Implementors represent a specific kind of namespace (e.g., UTS, mount, user) and must provide the associated metadata and traversal methods required by NsFs and NsFile.

Required Methods§

Source

fn owner_user_ns(&self) -> Option<&Arc<UserNamespace>>

Returns the owner user namespace.

Every namespace is owned by the user namespace in which it was created. The only exception is the initial user namespace, which has no owner; in that case this method returns None.

Source

fn parent(&self) -> Result<&Arc<Self>, Error>

Returns the parent namespace, if one exists.

This is meaningful only for hierarchical namespace types (user and PID namespaces). Non-hierarchical namespaces should return EINVAL.

Source

fn stashed_dentry(&self) -> &StashedDentry

Returns the dentry cache used to maintain the namespace’s NsFs entry.

Each namespace instance embeds a StashedDentry so that repeated lookups of its pseudo-filesystem path can reuse the same Dentry instead of allocating a new one every time.

Provided Methods§

Source

fn get_path(self: &Arc<Self>) -> Path

Returns the pseudo filesystem Path associated with this namespace.

The path is lazily created on first access and cached via StashedDentry for subsequent calls.

Dyn Compatibility§

This trait is not dyn compatible.

In older versions of Rust, dyn compatibility was called "object safety", so this trait is not object safe.
