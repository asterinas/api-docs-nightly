Module pseudofs 

Source
Expand description

The infrastructure of pseudo FSes along with some naive implementations of pseudo FSes.

Pseudo FSes are those that have no physical storage. Some pseudo FSes are trivial, e.g., PipeFs, SockFs, PidfdFs, and AnonInodeFs. Such trivial pseudo FSes are directly implemented in this module using a generic pseudo FS implementation called NaivePseudoFs.

On the other hand, some pseudo FSes are quite complex, including RamFs, ProcFs, and SysFs. Just to name a few. They are implemented in their own modules.

Pseudo FSes are not backed by physical storage devices, but they still need to be assigned container device IDs (see Metadata::container_dev_id). Linux’s behavior is to assign anonymous device IDs (whose major IDs are 0). As such, this module provides AnonDeviceId to acquire and recycle anonymous device IDs.
