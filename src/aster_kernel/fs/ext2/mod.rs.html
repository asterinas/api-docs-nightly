3//! A safe Rust Ext2 filesystem.
4//!
5//! The Second Extended File System(Ext2) is a major rewrite of the Ext filesystem.
6//! It is the predominant filesystem in use by Linux from the early 1990s to the early 2000s.
7//! The structures of Ext3 and Ext4 are based on Ext2 and add some additional options
8//! such as journaling.
9//!
10//! The features of this version of Ext2 are as follows:
11//! 1. No unsafe Rust. The filesystem is written is Rust without any unsafe code,
12//!    ensuring that there are no memory safety issues in the code.
13//! 2. Deep integration with PageCache. The data and metadata of the filesystem are
14//!    stored in PageCache, which accelerates the performance of data access.
15//! 3. Compatible with queue-based block device. The filesystem can submits multiple
16//!    BIO requests to be block device at once, thereby enhancing I/O performance.
17//!
18//! # Example
19//!
20//! ```no_run
21//! // Opens an Ext2 from the block device.
22//! let ext2 = Ext2::open(block_device)?;
23//! // Lookup the root inode.
24//! let root = ext2.root_inode()?;
25//! // Create a file inside root directory.
26//! let file = root.create("file", InodeType::File, FilePerm::from_bits_truncate(0o666))?;
27//! // Write data into the file.
28//! const WRITE_DATA: &[u8] = b"Hello, World";
29//! let len = file.write_at(0, WRITE_DATA)?;
30//! assert!(len == WRITE_DATA.len());
31//! ```
32//!
33//! # Limitation
34//!
35//! Here we summarizes the features that need to be implemented in the future.
36//! 1. Supports merging small read/write operations.
37//! 2. Handles the intermediate failure status correctly.
