3//! Per-operation FUSE request/reply definitions and encoders.
4//!
5//! Each submodule corresponds to one FUSE operation and owns both the on-wire
6//! structures and the `FuseOperation` implementation used to serialize a
7//! request and parse its reply. Types shared across operations (headers,
8//! `Attr`, `EntryReply`, directory entries) live in the crate root.
9//!
10//! Operation payload types use the `Req` suffix for request payloads sent to
11//! the server and the `Reply` suffix for reply payloads returned by the server.
12//!
13//! Operation wrapper types use the `Operation` suffix and implement [`crate::FuseOperation`].
